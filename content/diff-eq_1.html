Introduction¶

This iPython notebook was written to explore the various different numerical methods for solving first-order ODEs. Three methods are explored: Euler's Methods, The Midpoint Method, and the Classical Runge-Kutta method. RMS error is explored for each method.

Euler's Method:¶

Basic method, using the Taylor Series to approximate y(x) as a tangent line to the point (x,y(x)).

given dy/dx=f(x,y), Δy≈f(x,y)Δx⟹y(x+h)≈y(x)+h∗f(x,y(x))

In [2]:
def eulers_method(f, x, y, h):
    """
    Given a function f, the current x and the current y, returns an approximation of y(x+h) using Euler's method
    assuming that dy/dx = f(x,y)
    """
    return y + h * f(x,y)

Midpoint method:¶

Slightly more accurate than Euler's method, using the slope of the point (x+h/2,y(x+h/2)) (the midpoint of the interval) rather than (x,y(x)), as the midpoint better represents the averager slope on the interval [x,x+h].

y(x+h)≈y(x)+h∗f(x+h/2,y(x)+h/2∗f(x,y))

In [3]:
def midpoint_method(f, x, y, h):
    """
    Given a function f, the current x and the current y, returns an approximation of y(x+h) using the Midpoint method
    assuming that dy/dx = f(x,y)
    """
    return y + h * f(x+h/2.,y + h/2. * f(x,y))

Runge-Kutta Method:¶

The most roboust method, using a weighted average of 4 slopes (k1,⋯,k4) as the best estimate for the average slope on the interval.

k1k2k3k4=f(x,y(x))=f(x+h/2,y(x)+h/2∗k1)=f(x+h/2,y(x)+h/2∗k2)=f(x+h,y+h∗k3)

and y(x+h)≈y(x)+h/6∗(k1+2∗k2+2∗k3+k4)

In [4]:
def runge_kutta(f, x, y, h):
    """
    Given a function f, the current x and the current y, returns an approximation of y(x+h) using the Runge-Kutta method
    assuming that dy/dx = f(x,y)
    """
    k1 = f(x, y)
    k2 = f(x + h/2., y + h/2. * k1)
    k3 = f(x + h/2., y + h/2. * k2)
    k4 = f(x + h, y + h * k3)
    
    return y + h/6. * (k1 + 2*k2 + 2*k3 + k4)

Second Order ODEs¶

Let's extend our methods to solving ODEs in the form:

y′′=f(x,y(x),y′(x)),y(0)=y0,y′(0)=y′0

We can tackel this by re-writing it as a system of two first-order ODEs:

d(y′)/dxdy/dx=f(x,y(x),y′(x))=y′

For example, Euler's method would look like:

y′(x+h)y(x+h)≈y′(x)+h∗f(x,y(x),y′(x))≈y(x)+h∗y′(x)
In [10]:
def euler_2nd(f, x, y, yp, h):
    """
    Approximates the solution to y'' = f(x, y, yp) using an extended Euler's method
    
    Return is a 2-tuple containing an approximation of (y'(x+h), y(x+h))
    """
    
    return (y + h*yp,yp + h*f(x, y, yp))

Midpoint Method for Second Order ODEs¶

Consider d2y/dx2=f(x,y(x),y′(x)). Let u=(y(x),y′(x)). We then have du/dx=(y′(x),y′′(x))=(y′(x),f(x,u))

From this notation we can first re-derive Euler's Method for 2nd-order equations without handwaving. Namely u(x+h)≈u(x)+h∗u′(x) and thus (y(x+h),y′(x+h))≈(y(x)+h∗y′(x),y′(x)+h∗y′′(x))=(y(x)+h∗y′(x),y′(x)+h∗f(x,y(x),y′(x)) which was previously given without explanation, and broken up by components.

Now let's use this notation to find the midpoint formula for 2nd order equations. We have:

u(x+h)(y(x+h),y′(x+h))y(x+h)y′(x+h)y(x+h)y′(x+h)≈u(x)+h∗u′(x+h/2)≈(y(x),y′(x))+h∗(y′(x+h/2),y′′(x+h/2))∴≈y(x)+h∗y′(x+h/2)≈y′(x)+h∗f(x+h/2,y(x+h/2),y′(x+h/2))∴≈y(x)+h∗(y′(x)+h/2∗f(x,y(x),y′(x)))≈y′(x)+h∗f(x+h/2,y(x)+h/2∗y′(x),y′(x)+h/2∗f(x,y(x),y′(x)))

which addmitedly looks scary but luckily Python is powerful. Let's start by implementing this.

def midpoint_2nd(f, x, y, yp, h):
    """
    Approximates the solution to y'' = f(x, y, yp) using an extended midpoint method
    
    Return is a 2-tuple containing an approximation of (y'(x+h), y(x+h))
    """
    
    yn = y + h*(yp + h/2 * f(x,y,yp))
    ypn = yp + h*f(x+h/2,y+h/2*yp,yp+h/2*f(x,y,yp))
    
    return (yn, ypn)

Runge-Kutta for Second Order ODEs¶

We'll start by taking the first-order Runge-Kutta and swapping the y's for u=(y,y′):

k1k2k3k4=(y′(x),f(x,u(x)))=(y′(x)+h/2∗k1[1],f(x+h/2,u(x)+h/2∗k1))=(y′(x)+h/2∗k2[1],f(x+h/2,u(x)+h/2∗k2))=(y′(x)+h∗k3[1],f(x+h,u(x)+h∗k3))

and u(x+h)≈u(x)+h/6∗(k1+2∗k2+2∗k3+k4)

Notice the ki's are all 2-tuples now. Let's let the + and ∗ operators be element-wise. Further ki[j] is the jth element of ki, indexed starting at 0. Whereas in the orignal Runge-Kutta method ki was a guess for the average y′ on [x,x+h], ki is now a guess for the average (y′(x),y′′(x)) on the same interval.

def runge_kutta_2nd(f, x, y, yp, h):
    """
    Approximates the solution to y'' = f(x, y, yp) using an extended Runge-Kutta method
    
    Return is a 2-tuple containing an approximation of (y'(x+h), y(x+h))
    """
    
    k1 = (yp, f(x, y, yp))
    k2 = (yp + h/2*k1[1], f(x+h/2, y + h/2*k1[0], yp + h/2*k1[1]))
    k3 = (yp + h/2*k2[1], f(x+h/2, y + h/2*k2[0], yp + h/2*k2[1]))
    k4 = (yp + h*k3[1], f(x+h, y+h*k3[0], yp + h*k3[1]))
          
    return np.array((y,yp)) + h/6*(np.array(k1) + 2*np.array(k2) + 2*np.array(k3) + np.array(k4))

Conclusion¶

In this document, I have explored three popular techniques for solving first-order differential equations. I then extended each of the techniuqes to solving second order ODEs. If the three, Runge-Kutta is by far the most accurate, though admitteddly more complicated. The second order methods should be extendable to 3rd and higher-order ODEs.

Higher Differential Equations¶

Let's just consider what Runge-Kutta would look like for nth order ODEs. In particular, we want to solve:

dnydxn=f(x,y(x),y′(x),⋯,y(n−1)(x))y(0)=y0,⋯,y(n−1)(0)=y(n−1)0

Let u=(y(x),y′(x),⋯,y(n−1)(x)) and our differential equation becomes du/dx=(y′(x),⋯,y(n)(x),f(x,u(x))).

Euler's method would look like:

y(n−1)(x+h)y′′(x+h)y′(x+h)y(x+h)≈y(n−1)(x)+h∗f(x,u(x))⋮≈y′′(x)+h∗y′′′(x)≈y′(x)+h∗y′′(x)≈y(x)+h∗y′(x)

which as always is pleasingly simple yet terrible in practice. It is, however, a good starting place. Let's now generalize the 2nd-order Runge-Kutta. k1 is easy because it is equivilent to Euler's method. The rest will come from generalizing the expression the 2nd-order method, noting that ki[n] is our ith estimate for E[y(n)(x)] on the interval [x,x+h].

k1k2k3k4=(y′(x),y′′(x),⋯,y(n−1)(x),f(x,u(x)))=(y′(x)+h/2∗k1[1],⋯,y(n−1)(x)+h/2∗k1[n−1],f(x+h/2,u(x)+h/2∗k1))=(y′(x)+h/2∗k2[1],⋯,y(n−1)(x)+h/2∗k2[n−1],f(x+h/2,u(x)+h/2∗k2))=(y′(x)+h∗k3[1],⋯,y(n−1)(x)+h∗k3[n−1],f(x+h,u(x)+h∗k3))

and as always, u(x+h)≈u(x)+h/6∗(k1+2∗k2+2∗k3+k4).

To test this claim, let's try a 3rd-order ODE: d3ydx3=−y′′(x)−25y′(x)−25y(x)

Final Result¶

Here's my final result. Given a function f(x,u), this uses Runge-Kutta to solve du/dx=f(x,u) with initial conditions for (u[0](0),⋯,u[n−1](0))=u0, with u[i](x)=y(i)(x). As such, any nth order differential equation with initial conditions can be solved with way.

In particular inputting f, x, h, and u(x), this method returns an approximation for u(x+h)

throws ClassNotFoundException, InstantiationException, IllegalAccessException, UnsupportedLookAndFeelExceptiondef runge_kutta_nth(f, x, u, h):
    """
    Uses Runge-Kutta to approximate du/dx = f(x, u), with u = (y, y', ..., y^(n-1)).
    
    Given u(x), returns an approximation for u(x + h).
    """
    n = len(u)
    
    k1 = np.zeros(n)
    k2 = np.zeros(n)
    k3 = np.zeros(n)
    k4 = np.zeros(n)

    
    for i in range(0,n-1):
        k1[i] = u[i+1]
    k1[n-1] = f(x, u)
    
    for i in range(0, n-1):
        k2[i] = u[i+1] + h/2. * k1[i+1]
    k2[n-1] = f(x + h/2., np.array(u) + h/2. * k1)
    
    for i in range(0, n-1):
        k3[i] = u[i+1] + h/2. * k2[i+1]
    k3[n-1] = f(x + h/2., np.array(u) + h/2. * k2)
    
    for i in range(0, n-1):
        k4[i] = u[i+1] + h * k3[i+1]
    k4[n-1] = f(x + h, np.array(u) + h * k3)
    
    return u + h/6. * (k1 + 2*k2 + 2*k3 + k4)

u0 = [1,.075,0]                                   #Initial Conditions
 
h = .01
x0 = 0
x_max = 10

n = int(ceil((x_max - x0)/h))

x = np.zeros(n)
u = np.zeros([n,len(u0)]) 
u[0] = u0 

for i in range(1,n):
    x[i] = x[i-1] + h
    u[i] = runge_kutta_nth(f, x[i-1], u[i-1], h)
